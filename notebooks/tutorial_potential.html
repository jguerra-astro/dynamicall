gNFW#

\[ \rho(r)=\frac{\rho_0}{\left(\frac{r}{r_s}\right)^\gamma \left[1+\frac{r}{r_s}\right]^{3-\gamma}} \]

Binned Power Laws#

If you’ve used Gravsphere you may want to use the same scheme for the potential. Its essentially a broken power law with more breaks then usual and without the smooothing factor as in the Hernquist-Zhao Model.

\[\begin{split} \rho(r) = \rho_{s} \begin{cases} \left(\frac{r}{0.25R_{1/2}}\right)^{\gamma} & r \lt 0.25 R_{1/2} \\ \left(\frac{r}{0.5R_{1/2}}\right)^{\beta} \left(\frac{0.5 R_{1/2}}{0.25 R_{1/2}}\right)^{\beta} & 0.25 R_{1/2} \le r \lt 0.5 R_{1/2} \\ \left(\frac{r}{R_{1/2}}\right)^{\delta} \left(\frac{R_{1/2}}{0.5 R_{1/2}}\right)^{\delta} \left(\frac{0.5 R_{1/2}}{0.25 R_{1/2}}\right)^{\beta} & 0.5 R_{1/2} \le r \lt R_{1/2} \\ \left(\frac{r}{2R_{1/2}}\right)^{\sigma} \left(\frac{2 R_{1/2}}{R_{1/2}}\right)^{\sigma} \left(\frac{R_{1/2}}{0.5 R_{1/2}}\right)^{\delta} \left(\frac{0.5 R_{1/2}}{0.25 R_{1/2}}\right)^{\beta} & R_{1/2} \le r \lt 2 R_{1/2} \\ \left(\frac{r}{4R_{1/2}}\right)^{\alpha} \left(\frac{4 R_{1/2}}{2 R_{1/2}}\right)^{\alpha} \left(\frac{2 R_{1/2}}{R_{1/2}}\right)^{\sigma} \left(\frac{R_{1/2}}{0.5 R_{1/2}}\right)^{\delta} \left(\frac{0.5 R_{1/2}}{0.25 R_{1/2}}\right)^{\beta} & r \ge 2 R_{1/2} \\ \end{cases} \end{split}\]

Where i’ve explicitly left terms uncanceled to show the reason they’re there and to make it easier to show how to generalize it to more breaks inline with how the code is written.

binnedpl = models.PowerLaws(
                rhos = 1e7,
                R_half = 1.0,
                gamma = -1.0,
                beta  = -2.0,
                delta = -3.0,
                sigma = -4.0,
                alpha = -5.0)

fig,ax = plt.subplots()
r = jnp.logspace(-2,1,100)
ax.plot(r,jax.vmap(jax.jit(binnedpl.density))(r))

ax.set(
    xscale='log',
    yscale='log',
    xlabel='r [kpc]',
    ylabel=r'$\rho$ [M$_{\odot}$ kpc$^{-3}$]',
);

Custom Potentials#

It’s possible that one of the models you need are not implemented in the library. You can raise an issue on the github page and we’ll try to implement it as soon as possible. But in the meantime, you can define your own potential using the base class JaxPotential

Almost every method is implemented numerically in the base class so its possible to define all methods in the class with a few starting functions. The easiest is to define the density.

We’ll use the Plummer model as an example. Since a lot of the functions associated with this model are analytic, it’ll be a good way to test the accuracy of the our numerical methods. If they’re not up to par we can always tweak certain default parameters in the base class inorder to get results we’re satified with.

from dynamicAll.base import JaxPotential

class CustomPot(JaxPotential):
    def __init__(self,M,a):

        self.M = M
        self.a = a
        self.params = {'M':M,'a':a}

    def density(self,r):
        return CustomPot._density(r,self.params)
    
    def _density(r,params):
        q = r / params['a']
        coeff = 3 * params['M'] / (4 * jnp.pi * params['a']**3)

        return coeff * (1 + q**2) ** (-5 / 2)

Getting better performance#

The default parameters are written to give an accurate solution off the bat, but theyre overkill for most cases. Given we care about performance, our model should be as good as it needs to be and no better (as that will tend to come with a performance hit).

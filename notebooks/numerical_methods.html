CustomPotential#

Theres a base class called JaxDensity where all of the numerical methods are defined that can be found in base.py. Specific sublasses are defined for each model in models.py

If a dynamical function is known analytically it is written in the subclass, otherwise the default is to use the numerical methods defined in the base class.

For testing purposes it maybe useful to compare the numerical methods to the analytical methods to see if they’re working correctly.

%load_ext autoreload
%autoreload 2

from dynamicAll.base import JaxDensity

The general idea if you want to implement a potential that is not built in is that all you have to define is one or two functions and then everything else can be calculated numerically.

The more functions you specify will lead to slightly faster calculations and perhaps slighly more accurate results, but we show that even defining just one function you can achieve good results.

We’ll start with the Plummer model as an example.

A particularly tricky function to calculate given just a density profile is the Distribution function. This is a function that gives the probability of finding a star with a given energy at a given position. It is defined as:

\[ f(\mathcal{E}) = \frac{1}{\sqrt{8}\pi^2} \frac{d}{d\mathcal{E}}\int_0^{\mathcal{E}}\frac{d\rho}{d\psi}\frac{d\psi}{\sqrt{\mathcal{E}-\psi}} \]

The combination of integrals and derivatives can make this a difficult function to calculate. Below we show how we can calculate this function for the Plummer model.

from functools import partial
import jax
import jax.numpy as jnp


class TestPotential(JaxDensity):
    def __init__(self, M: float, rs: float):
        self._M = M
        self._rs = rs
        self._params = {"M": self._M, "rs": self._rs}

    @classmethod
    @partial(jax.jit, static_argnums=(0,))
    def _density(cls, r: jnp.ndarray, params) -> jnp.ndarray:
        """
        Parameters
        ----------
        r : np.ndarray
            radius : np.sqrt(x**2+y**2 + z**2)
        M : float
            _description_
        a : float
            _description_

        Returns
        -------
        np.ndarray
            _description_
        """
        q = r / params["rs"]
        coeff = 3 * params["M"] / (4 * jnp.pi * params["rs"] ** 3)

        return coeff * (1 + q**2) ** (-5 / 2)

params = {"M": 3e5, "rs": 0.5}

test = TestPotential(**params)

It could be seen as annoying to have to physically vectorize functions, but it does give you a lot of control over how the functions are implemented and how they’re used. If you wanted to see the potential for wide range of parameters you could also vectorize over that variable

params_list = [{"M": 1.0, "rs": 1.0}, {"M": 1.0, "rs": 2.0}, {"M": 1.0, "rs": 3.0}]
params_dict = {k: jnp.array([p[k] for p in params_list]) for k in params_list[0]}

vectorized_two = jax.vmap(
    jax.vmap(TestPotential._mass, in_axes=(0, None)),
    in_axes=(None, 0),
)

test_vectorization = vectorized_two(r, params_dict)

test_vectorization.shape

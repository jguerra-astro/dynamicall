class dynamicAll.models.Anisotropy[source]#

Bases: object

Base class for velocity anisotropy profiles

The anistropy parameter \(\beta(r)\) is defined as

\[\beta(r) = 1-\frac{\sigma_{t}^{2}}{\sigma_{r}^{2}}\]

where \(\beta\) is the anisotropy parameter, \(\sigma_{r}\) is the radial velocity dispersion, and \(\sigma_{t}\) is the tangential velocity dispersion which define as \(\sigma_{t}^{2} = \sigma_{\theta}^{2} + \sigma_{\phi}^{2}\). It is a measure of the degree of radial vs tangential orbits in a system. With line of sight velocities, the anisotropy parameter is not directly observable and poorly constrained.

\(\beta=0\) corresponds to isotropic orbits, \(\beta< 0\) corresponds to tangentially biased orbits, and \(\beta>0\) corresponds to radially biased orbits. \(\beta=1\) corresponds to purely radial orbits. \(\beta=-\infty\) corresponds to purely tangential orbits.

class dynamicAll.models.BetaSymmetric(beta_tilde)[source]#

Bases: Anisotropy

This is the symmetric version of a constant anisotropy profile. Define by

\[\tilde{\beta} = \frac{\beta}{2-\beta}\]

or equivalently

\[\beta = \frac{2\tilde{\beta}}{1+\tilde{\beta}}.\]

This allows for a simple prior on \(\tilde{\beta}\) (-1,1) that maps to \(\beta\) (-infinity,1). and \(\tilde{\beta} = 0\) corresponds to \(\beta = 0\).

class dynamicAll.models.HernquistZhao(rhos: float, rs: float, a: float, b: float, c: float, rlim='r200')[source]#

Bases: JaxDensity

Hernquist-Zhao density profile

\[\rho(r) =\frac{\rho_s}{\left(\frac{r}{r_s}\right)^a \left[1+\left(\frac{r}{r_s}\right)^{b}\right]^{\frac{c-a}{b}}}\]
Parameters:

  • rhos (float) – scale density – units: [M L^-3] if mass density, [L^-3] for number density

  • rs (float) – scale radius – units: [L]

  • a (float) – inner slope – for \(\frac{r}{r_s} \ll 1 ~ \rightarrow \rho \sim r^{-a}\)

  • b (float) – characterizes width of transition between inner slope and outer slope

  • c (float) – outer slope – for \(r/r_s \gg 1~ \rho \sim r^{-c}\)

Notes

for \(c \ge 3\) mass diverges as \(r\rightarrow \infty\) therefore default priors on c go from 1 to 3. You can get around this, by limiting relevant integrations to a finite radius e.g. \(r_{200}\).

There are various testing functions that are useful for testing the accuracy of our integration scheme

References

Baes 2021 : http://dx.doi.org/10.1093/mnras/stab634

An & Zhao 2012 : http://dx.doi.org/10.1093/mnras/sts175

Zhao 1996 : http://dx.doi.org/10.1093/mnras/278.2.488

J_analytic(theta, D)[source]#
\[\mathrm{J(\theta | D,\rho_s,r_s)} =\frac{\pi \rho_0^2 r_{\mathrm{s}}^3}{3 D^2 \Delta^4}\left[2 y\left(7 y-4 y^3+3 \pi \Delta^4\right)\right.\left.+6\left(2 \Delta^6-2 \Delta^2-y^4\right) X(y)\right]\]

where \(y=D\theta/r_s\), \(\Delta=\sqrt{1-y^2}\) and

\[\begin{split}X(s)= \begin{cases}\frac{1}{\sqrt{1-s^2}} \operatorname{Arcsech} s, & 0 \leq s \leq 1 \\ \frac{1}{\sqrt{s^2-1}} \operatorname{Arcsec} s, & s \geq 1.\end{cases}\end{split}\]

This is using the approximation that \(d\Omega d\mathcal{l} =\frac{2πRdRdz}{D^{2}}\).
